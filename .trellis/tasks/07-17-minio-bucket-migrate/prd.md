# MinIO 桶迁移工具（复制）

## Goal

开发一个 MinIO 桶数据迁移（复制）工具，提供后端 API 和前端 Web UI，支持跨桶和跨集群的数据复制任务管理。

## Background / User Value

- MinIO 是兼容 S3 协议的对象存储
- 用户需要在不同 MinIO 桶之间、或不同 MinIO 集群之间复制数据
- 需要一个可视化的管理界面来创建、监控和管理迁移任务

## Confirmed Facts（来自用户初始描述）

- 需要**后端**：负责与 MinIO（S3 兼容 API）交互，执行数据复制
- 需要**前端**：提供 Web UI 管理迁移任务
- 支持**跨桶**复制（同一集群内不同桶）
- 支持**跨集群**复制（不同 MinIO 集群之间）
- 本质是**复制**工具（非移动/剪切）

## Technical Decisions

- **后端语言**: Python（用户确认）
- **后端框架**: FastAPI（本小姐推荐，异步支持好，自动生成 OpenAPI 文档）
- **前端框架**: React + TypeScript + Ant Design（用户确认）
- **数据库**: SQLite + SQLAlchemy（用户确认）
- **部署模式**: 独立 Web 服务 + Docker 管理（用户确认）
- **迁移模式**: 手动触发全量复制（用户确认）
- **迁移范围**: 整个桶，不支持按前缀过滤（用户确认）
- **用户认证**: 需要登录（`.env` 配置默认帐密 `admin/admin123456`，登录后可修改密码）（用户确认）

## Requirements

### Backend

- [ ] MinIO/S3 连接管理（源端和目标端配置）
- [ ] 桶列表浏览
- [ ] 对象复制：支持跨桶、跨集群
- [ ] 迁移任务创建、状态跟踪、日志记录
- [ ] 并发控制与限速
- [ ] 用户认证（JWT 登录，密码修改）

### Frontend

- [ ] 登录页面
- [ ] MinIO 连接配置管理
- [ ] 源/目标桶选择界面
- [ ] 迁移任务仪表盘（状态、进度、历史）
- [ ] 任务详情与日志查看
- [ ] 迁移任务手动触发/取消/重试
- [ ] 密码修改

## Acceptance Criteria

- [ ] 能配置 MinIO 源端和目标端连接（endpoint、access key、secret key、bucket）
- [ ] 能从源桶复制对象到目标桶（同一集群内）
- [ ] 能跨 MinIO 集群复制对象
- [ ] Web UI 上能创建、查看、取消迁移任务
- [ ] 迁移任务显示实时进度
- [ ] Web UI 需要登录才能访问，默认帐密 `admin/admin123456`
- [ ] 登录后可修改密码

## Out of Scope（初始）

- MinIO 集群部署和管理
- S3 兼容的其他对象存储（如 AWS S3）— 先聚焦 MinIO
- 实时双向同步
- 按前缀/路径过滤迁移
- 定时调度迁移任务
